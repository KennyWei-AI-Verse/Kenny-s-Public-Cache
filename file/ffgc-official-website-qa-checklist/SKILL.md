# FFGC 官网 QA Checklist Skill

## 目标

对 FFGC 官网进行上线前快速 QA，发现：

- 功能 BUG
- 页面展示异常
- 移动端适配问题
- 视觉粗糙 / 比较丑 / 不统一的地方
- 链接、按钮、跳转错误
- 明显性能或加载问题

测试站点：

https://ffgcoffical.run.ingarena.net/

> 注意：竞猜活动还没做，本轮不测竞猜功能，只记录入口是否需要隐藏 / 禁用 / 标注 Coming Soon。

---

## 测试原则

1. 优先从真实用户视角测试，不只看接口是否 200。
2. 每个问题都要有截图或录屏证据。
3. 不要只写“有点怪”，要说明：页面位置、触发步骤、实际表现、期望表现、严重程度。
4. 重点覆盖 PC + Mobile。
5. 视觉问题也要记录，尤其是：字太小、间距不舒服、按钮不明显、图片拉伸 / 裁切怪、中英文混排不统一、深浅色对比不足。

---

## 测试环境

至少覆盖：

- Chrome 桌面端
- 手机尺寸模拟：iPhone 13 / 14、Android 常见尺寸
- 网络：正常网络；慢速 4G 简单观察首屏加载

---

## 一、首页检查

### 基础展示

- [ ] 首页能正常打开，无白屏 / 报错
- [ ] 首屏加载后核心信息清楚
- [ ] Logo / 主视觉 / Banner 无变形
- [ ] 页面标题、活动名称、时间信息无明显错误
- [ ] 无乱码、占位符、测试文案
- [ ] 无英文 pseudo text / lorem ipsum
- [ ] 无 console 明显报错

### 视觉体验

- [ ] 首屏视觉是否够正式、像官方赛事官网
- [ ] 背景、文字、按钮对比度是否足够
- [ ] 字号层级是否清楚
- [ ] 模块之间间距是否舒服
- [ ] 是否有明显“很空”或“很挤”的区域
- [ ] 动效是否自然，不突兀、不掉帧

---

## 二、导航与跳转

- [ ] 顶部导航每一项都能点击
- [ ] 点击后能跳到正确区域或页面
- [ ] 当前页面 / 当前模块状态清楚
- [ ] 移动端菜单可正常展开 / 收起
- [ ] 移动端菜单不会遮挡内容
- [ ] 返回顶部 / 锚点跳转位置正确
- [ ] 外链能正常打开
- [ ] 无 404 / 空页面 / 错误跳转

---

## 三、核心内容模块

### 赛事信息

- [ ] 赛事名称正确
- [ ] 时间、赛程、阶段信息正确
- [ ] 地区 / 队伍 / 规则信息展示清楚
- [ ] 信息层级容易理解
- [ ] 移动端不会横向溢出

### 队伍 / 选手 / 分组

- [ ] 队伍 Logo 正常显示
- [ ] 队名无截断异常
- [ ] 图片比例统一
- [ ] 卡片 hover / 点击状态正常
- [ ] 空数据状态合理
- [ ] 排列在移动端自然换行

### 赛程 / 比分 / 排名

- [ ] 日期、时间、队伍、比分对齐清楚
- [ ] 未开始 / 进行中 / 已结束状态区分清楚
- [ ] 排名顺序展示合理
- [ ] 长队名不会挤爆布局
- [ ] 表格或列表在手机端可读

### 新闻 / 公告 / 内容区

- [ ] 标题、摘要、封面图正常
- [ ] 点击进入详情正常
- [ ] 详情页排版正常
- [ ] 图片不糊、不变形
- [ ] 时间格式统一
- [ ] 无空封面 / broken image

### 视频 / 直播 / 社媒入口

- [ ] 播放入口 / 跳转入口正常
- [ ] 图标和平台名称清楚
- [ ] 外链打开正确
- [ ] 移动端点击区域足够大
- [ ] 没有失效链接

---

## 四、竞猜活动特殊处理

由于竞猜活动尚未完成：

- [ ] 如果页面有竞猜入口，确认是否需要隐藏
- [ ] 如果保留入口，应有明确状态：Coming Soon / 敬请期待 / 活动暂未开放
- [ ] 点击竞猜入口不应进入空白页 / 报错页
- [ ] 不应让用户误以为现在可以参与竞猜

建议记录：

```text
竞猜入口当前状态：
- 是否出现：
- 点击结果：
- 建议：隐藏 / 禁用 / 加 Coming Soon / 保留
```

## 五、移动端重点检查

- [ ] 首屏是否一眼看得懂
- [ ] 顶部导航不遮挡内容
- [ ] 横向无溢出
- [ ] 按钮点击区域足够大
- [ ] 卡片排列不混乱
- [ ] 字体不小到难读
- [ ] 图片裁切合理
- [ ] 页面滚动顺畅
- [ ] 底部内容没有被遮住
- [ ] 弹窗 / 菜单 / 浮层可关闭

---

## 六、视觉优化 Checklist

重点找“比较丑”的地方：

- [ ] 模块风格是否统一
- [ ] 卡片圆角、阴影、边框是否一致
- [ ] 主按钮和次按钮层级是否清楚
- [ ] 颜色是否过多或不协调
- [ ] Banner 是否像正式赛事页面
- [ ] 是否有明显廉价感素材
- [ ] 图片是否糊、压缩严重、比例不对
- [ ] 标题是否太长导致换行难看
- [ ] 中英文 / 数字 / 符号间距是否舒服
- [ ] 页脚是否完整、干净

---

## 七、性能与稳定性

- [ ] 首屏加载是否明显过慢
- [ ] 图片是否逐张慢慢闪出
- [ ] 页面滚动是否卡顿
- [ ] 切换模块是否有明显延迟
- [ ] 刷新页面后状态正常
- [ ] 直接打开子页面 URL 正常
- [ ] 弱网下至少能看到基础内容
- [ ] 无明显 JS 报错

---

## 八、SEO / 分享基础检查

- [ ] 页面 title 合理
- [ ] 分享预览标题合理
- [ ] 分享预览图正常
- [ ] favicon 正常
- [ ] 页面描述不为空
- [ ] 无测试环境文案暴露给用户

---

## 九、问题严重程度

### P0 阻塞

- 页面打不开
- 白屏
- 主流程无法访问
- 大面积资源加载失败

### P1 严重

- 核心模块错误
- 重要跳转错误
- 移动端严重错位
- 用户会误解活动状态

### P2 一般

- 局部展示问题
- 图片裁切问题
- 文案错误
- 样式不统一
- 轻微交互问题

### P3 优化

- 视觉可更好
- 动效可优化
- 间距 / 字号 / 层级微调
- 体验建议

---

## 十、问题反馈模板

每个问题按这个格式记录：

```text
【严重程度】P1 / P2 / P3

【页面 / 模块】
例如：首页 Banner / 移动端导航 / 赛程模块

【问题描述】
简短说明看到的问题。

【复现步骤】
1. 打开 xxx
2. 点击 xxx
3. 看到 xxx

【实际结果】
现在是什么样。

【期望结果】
应该是什么样。

【设备 / 浏览器】
例如：iPhone 14 Safari / Chrome Desktop

【截图 / 录屏】
附截图或录屏。

【建议】
如果是视觉问题，给出具体优化建议。
```
